# Lesson-Manager (Лабораторні роботи №1-2)

**Lesson-Manager** — це навчальний проєкт для обліку навчальних предметів та розкладу занять. Проєкт демонструє еволюцію застосунку від консольного інтерфейсу до повноцінного графічного UI з використанням сучасних архітектурних підходів.

---

## 🏗️ Архітектура рішення
Проєкт розділений на декілька логічних шарів (проєктів), що забезпечує дотримання принципу **Single Responsibility**:

1.  **`LessonManager.Models`** — базові сутності (Subject, Lesson) без логіки відображення.
2.  **`LessonManager.Views`** — об'єкти представлення з обчислюваними полями (тривалість заняття, сумарний час) та логікою зв'язків.
3.  **`LessonManager.Services`** — логіка роботи з даними. Містить інтерфейс `IDataService` та `MockStorage` зі штучними даними.
4.  **`LessonManager.WPF`** — графічний інтерфейс користувача (Лабораторна робота №2).
5.  **`LessonManager.ConsoleApp`** — консольний інтерфейс (Лабораторна робота №1).

---

## 🛠️ Лабораторна робота №2: UI та IoC

У другій частині проєкту фокус був зміщений на розробку UI та впровадження архітектурних принципів **Inversion of Control (IoC)**.

### 1. Графічний інтерфейс (WPF)
Реалізовано систему навігації всередині одного вікна без відкриття нових вікон:
* **SubjectsListPage**: відображає список усіх предметів (сутностей 1-го рівня).
* **SubjectDetailsPage**: показує детальну інформацію про предмет та перелік усіх пов'язаних занять (сутностей 2-го рівня).
* **LessonDetailsPage**: детальний перегляд обраного заняття, включаючи його тривалість.

### 2. Dependency Injection (DI)
Для керування залежностями використано пакет `Microsoft.Extensions.DependencyInjection`:
* Реєстрація сервісів відбувається в `App.xaml.cs`.
* Сторінки отримують доступ до даних виключно через інтерфейс `IDataService`, що забезпечує гнучкість системи.

### 3. Навігація
* Використано компонент `Frame` для зміни контенту.
* Реалізовано можливість повернення назад на попередні екрани через `NavigationService.GoBack()`.
