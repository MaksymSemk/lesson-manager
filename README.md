# Lesson-Manager (Лабораторні роботи №1-2)

**Lesson-Manager** — це навчальний проєкт для обліку навчальних предметів та розкладу занять. Проєкт демонструє еволюцію застосунку від консольного інтерфейсу до повноцінного графічного UI з використанням сучасних архітектурних підходів.

---

## 🏗️ Архітектура рішення (MVVM + Repository)
Проєкт розділений на 3 основні рівні згідно з вимогами Лабораторної роботи №3:

1.  **Repositories (Рівень сховища)**: Містить інтерфейси `ISubjectRepository`, `ILessonRepository` та їх реалізації для роботи з `MockStorage`. Відповідає за доступ до "сирих" даних.
2.  **Services (Рівень бізнес-логіки)**: Сервіс `IDataService` тепер працює виключно через репозиторії (Dependency Injection).
3.  **ViewModels & UI (Рівень представлення)**: 
    * Повна реалізація патерну **MVVM**.
    * Файли `.xaml.cs` містять лише ініціалізацію та задання `DataContext`.
    * Весь зв'язок між UI та логікою реалізовано через **Data Binding**.

---

### 🚀 Функціональні можливості (UI)
 1. Графічний застосунок побудований на базі WPF і реалізує наступну логіку:

2. Головна сторінка: Перегляд списку всіх предметів (сутність 1-го рівня), отриманих із сервісу.

3. Деталі предмета: Відображення повної інформації про предмет та списку всіх пов'язаних занять (сутності 2-го рівня).

4. Деталі занять: Детальний опис конкретного заняття з автоматичним розрахунком його тривалості.
---
### ⚙️ Технічні особливості
1. Dependency Injection (DI): Використано IoC-контейнер Microsoft.Extensions.DependencyInjection для реєстрації та впровадження сервісів.

2. Навігація: Реалізована через зміну сторінок (Page) всередині одного Frame у MainWindow, що виключає відкриття зайвих вікон.

3. Inversion of Control: Вся взаємодія між UI та даними відбувається через інтерфейси, що відповідає принципу Dependency Inversion..
