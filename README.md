# Lesson-Manager: Лабораторна робота №1

## Опис проєкту
**Менеджер занять** — це застосунок для обліку предметів і занять, що належать до цих предметів. Кожен предмет логічно пов'язаний із набором занять. Проєкт реалізований як консольний застосунок із використанням принципів об'єктно-орієнтованого програмування та багатошарової архітектури.

---

## 🏗️ Архітектура та структура рішення
Згідно з принципами **Single Responsibility** та вимогами до проєктування, рішення розділене на наступні проєкти:

* **`LessonManager.Models`**: Бібліотека класів для зберігання даних. Містить "чисті" моделі без обчислюваних полів та колекцій.
* **`LessonManager.Views`**: Бібліотека класів, що відповідають за представлення даних, їх створення та редагування. Тут реалізовані обчислювані поля та зв'язки між сутностями.
* **`LessonManager.Services`**: Бібліотека, що містить сервіс для роботи зі сховищем та штучне сховище даних.
* **`LessonManager.ConsoleApp`**: Консольний застосунок для безпосередньої взаємодії з користувачем.

---

## 📊 Опис сутностей

### 1. Предмет (Сутність першого рівня)
* **Дані зберігання**: Унікальний ідентифікатор, назва, кількість кредитів ECTS та сфера знань (перечислення KnowledgeArea).
* **Представлення (View)**: Додатково містить колекцію пов'язаних занять та обчислюване поле загальної тривалості всіх занять.

### 2. Заняття (Сутність другого рівня)
* **Дані зберігання**: Ідентифікатор заняття, ID предмета-власника, дата, час початку та завершення, тема та тип заняття (перечислення LessonType).
* **Представлення (View)**: Додатково містить обчислюване поле тривалості конкретного заняття.

---

## 💾 Штучне сховище (Mock Storage)
Для тестування застосунку реалізовано клас сховища, який при створенні заповнюється наступними даними:
* **3 екземпляри** сутностей першого рівня (предмети) з різними даними.
* **12 екземплярів** сутностей другого рівня (заняття).
* Дотримано розподіл: **10 занять** належать одному предмету, а **2 заняття** — іншому.

---

## 🖥️ Логіка роботи застосунку
Після запуску програма виконує наступні кроки:
1.  Отримує колекцію предметів через сервіс роботи зі сховищем.
2.  Виводить список доступних предметів користувачу.
3.  Надає можливість вибрати конкретний предмет для перегляду деталей.
4.  Відображає повну інформацію про предмет та всі пов'язані з ним заняття.
5.  Дозволяє повернутися до головного меню або штатно завершити роботу.

---

## 🛠️ Технічні вимоги та обмеження
* Об'єкти зберігання першого рівня не містять колекцій об'єктів другого рівня.
* Об'єкти другого рівня не мають посилань на об'єкти першого рівня (зв'язок тільки через ID).
* Ідентифікатори сутностей захищені від зміни після створення.
* Використано конструктори для ініціалізації об'єктів без дублювання коду.
